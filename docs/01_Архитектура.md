# Общая архитектура

## Три ключевых принципа

### 1. Data-driven (данные отделены от кода)
Все «цифры» игры живут в **ScriptableObject**-ах — это ресурсы в проекте, изменяются без перекомпиляции.
- Оружие → `WeaponDataSO` (+ наследники)
- Броня → `ArmorDataSO`
- Характеристики персонажа → `CharacterStatsSO`
- Один удар в комбо → `AttackActionSO`

### 2. EventBus — никаких прямых ссылок между системами
Вместо `systemA.DoSomething()` используется `EventBus.RaiseEvent<IHandlerName>(h => h.OnMethod(...))`.
Подписаться = унаследовать интерфейс от `IGlobalSubscriber` и вызвать `EventBus.Subscribe(this)` в `OnEnable`.
Подробнее → [[Systems/EventBus]]

### 3. Стейт-машина на персонаже
`HeroController` держит экземпляры всех состояний. Каждое состояние знает своего владельца и управляет логикой одного поведения.
Подробнее → [[Systems/StateMachine]]

## Zenject (DI)

Единственный инсталлер: `Assets/Scripts/Installers/GameInstaller.cs`

| Биндинг | Тип |
|---|---|
| `InputService` | `IInitializable`, `IDisposable` — запускается автоматически |
| `InventoryService` | Single (plain class, без MonoBehaviour) |

> **Правило**: новые сервисы без MonoBehaviour биндятся здесь. MonoBehaviour-компоненты добавляются на GameObject персонажа напрямую.

## Компоненты на GameObject персонажа

Все `MonoBehaviour`-компоненты вешаются на одного и того же GameObject:

| Компонент                     | Пространство имён    |
| ----------------------------- | -------------------- |
| `HeroController`              | `Character`          |
| `HealthComponent`             | `Combat.Runtime`     |
| `StaminaComponent`            | `Combat.Runtime`     |
| `ComboController`             | `Combat.Controllers` |
| `BlockController`             | `Combat.Controllers` |
| `WeaponHandler`               | `Combat.Controllers` |
| `EquipmentController`         | `Combat.Controllers` |
| `CharacterController` (Unity) | встроен              |
| `Animator`                    | встроен              |

`HeroController.Awake()` ищет все эти компоненты через `GetComponent<>`.

## Форматы данных

### DamageInfo (struct)
Передаётся по всей боевой цепочке:
```
Amount        — итоговый урон
Type          — Physical / Magic
WeaponCategory — MeleePhysical / MeleeMagic / RangedPhysical / RangedMagic
Attacker      — GameObject источника урона
ComboIndex    — 0..3, для диагностики
```

### BlockResult (struct)
Результат попытки заблокировать удар:
```
Outcome      — NotBlocking / Parried / Blocked / BlockBroken
DamagePassed — сколько урона прошло сквозь блок
StaminaSpent — сколько стамины потрачено
```
